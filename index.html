The Solution

Pure-play infrastructure layer for peer-to-peer mobile data exchange

How it works:

  • Users list unused data they want to swap
  • Others match with compatible offers
  • Data is transferred directly via telco-provided USSD or in-app transfer menus
  • Both parties confirm completion

Key design principles:

  • Intra-network swaps at launch (MTN‚ÜîMTN, Airtel‚ÜîAirtel, etc.)
  • No wallet, no escrow, no custody
  • No resale of telecom services
  • Platform only facilitates discovery, matching, and confirmation

This architecture dramatically reduces regulatory friction and accelerates time-to-market.

Telco Compatibility

Designed to work with telcos, not against them

Data Swap does not touch telco infrastructure. Transfers remain user-initiated using telco-approved flows (USSD or in-app transfer menus). We only provide the marketplace discovery and confirmation layer.

  • User-Directed Execution: we provide the ‚Äúmap,‚Äù but the user holds the ‚Äúkeys.‚Äù We do not trigger transfers on a user‚Äôs behalf.
  • No resale: we do not sell or custody data or funds
  • No network access: we don‚Äôt require telco APIs or backend integration
  • NDPA/NDPC aligned: we do not store phone numbers embedded in USSD strings or transfer menus.

Instruction mapping: USSD/menu steps can change by network. Data Swap stays code-agnostic and updates the in-app instruction set instantly ‚Äî without changing the underlying platform.

Telco-aligned positioning: Data Swap increases perceived value of staying on a network (intra-network liquidity), supports retention (‚ÄúSIM stickiness‚Äù), and reduces churn by helping users recover value from leftover bundles.

Data Validity & Expiry

Expiry rules stay exactly the same

Data Swap does not reset, extend, or modify data validity. Any data exchanged retains the original expiry date set by the telecom operator.

Example: If a user transfers 3GB with 3 days remaining, the recipient still has 3 days to use it ‚Äî expiry does not change after transfer.

This keeps the platform fully aligned with telco policies and strengthens the ‚Äúno-resale / no-custody‚Äù architecture. We facilitate who uses the data, not how long it lasts.

Dispute Resolution

Closing the loop on ‚Äúhe said / she said‚Äù

Without telco APIs or escrow, disputes are handled with a simple, enforcement-first flow that protects honest users while keeping the platform regulatory-lite:

  • 1) Proof Upload: either party can upload a screenshot of ‚Äútransfer successful‚Äù (SMS / USSD menu / in-app confirmation).
  • 2) Reputation Strike: repeated disputes trigger rate-limits, temporary blocks, then removal.
  • 3) Automated Blacklisting: device + account signals reduce repeat bad-actor attempts.

Roadmap: OCR-style parsing can be introduced after we collect real-world proof formats across MTN/Airtel/Glo/9mobile to reduce false negatives.

Trust Stack

Defense-in-depth for a high-distrust environment

Data Swap does not try to ‚Äúrecover‚Äù lost data. Instead, we reduce loss frequency and make repeated abuse economically irrational using layered enforcement.

  • Economic friction: prepaid Swap Credits create cost for bad behavior and reduce payment failures.
  • Proof of execution: proof upload for flagged swaps; OCR verification is introduced after collecting real-world formats.
  • Hard stops: reputation strikes ‚Üí rate limits ‚Üí suspension; device/account blacklisting for repeat offenders.
